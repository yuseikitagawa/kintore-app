# バックエンド（スプレッドシートDB）デプロイ手順

HTMLアプリから読み書きする先の **Googleスプレッドシート＋Apps Script** を用意する。
一度やれば、あとはアプリ側の設定にURLを貼るだけ。所要 5分ほど。

> このデプロイだけは Google アカウントでの操作が必要（自動化できない部分）。

## 手順

### 1. スプレッドシートを作る
1. https://sheets.new を開く（ログインは `yusei.myostudio@gmail.com`）
2. 名前を「筋トレDB」などに変更（シートの中身は空でOK。Apps Scriptが自動でタブを作る）

### 2. Apps Script を貼る
1. メニューの **拡張機能 → Apps Script** を開く
2. 最初からある `function myFunction() {}` を全部消す
3. このフォルダの **`Code.gs`** の中身を全部コピーして貼り付け
4. 保存（💾 / Ctrl+S）

### 3. ウェブアプリとして公開（デプロイ）
1. 右上 **デプロイ → 新しいデプロイ**
2. 「種類の選択」の⚙️ → **ウェブアプリ** を選ぶ
3. 設定:
   - 説明: 任意（例: 筋トレAPI v1）
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
     （URLを知っている人だけがアクセス。認証なしでアプリから叩けるようにするため）
4. **デプロイ** を押す
5. 初回は認証を求められる → 自分のアカウントを選び、「詳細 → （安全ではないページ）に移動」→ 許可
6. 表示される **ウェブアプリのURL**（`https://script.google.com/macros/s/……/exec`）をコピー

### 4. アプリに URL を設定
1. `02_プロトタイプ/js/config.js` を開く
2. `API_URL: ''` の `''` の中にコピーしたURLを貼る
   ```js
   API_URL: 'https://script.google.com/macros/s/AKfy..../exec',
   ```
3. 保存してアプリ（index.html）を開き直す

これで、記録・メニュー編集がスプレッドシートに保存される。

## 動作確認
- ブラウザで `……/exec?type=all` を直接開くと、`{"menus":[...],"exercises":[...],"records":[...]}` が返る＝疎通OK
- アプリで「トレーニングを終了」を押す → スプレッドシートの「記録」タブに完了セットが行で増える
- メニュー編集で種目を足す/外す → 「種目マスタ」タブに反映される

## コードを直したら（再デプロイ）
`Code.gs` を編集したら **デプロイ → デプロイを管理 → 鉛筆✏️ → バージョン「新しいバージョン」→ デプロイ**。
URLは変わらないので config.js はそのままでよい。

## 仕組み（メモ）
- 読み取り: アプリが `GET ?type=all` でJSONを取得
- 書き込み: アプリが `POST`（`Content-Type: text/plain` にして preflight を回避）
- URL未設定でもアプリは動く（この端末の localStorage に保存）。あとからURLを入れれば以降がスプレッドシートに同期される
