# 筋トレ記録アプリ

「続けたくなる」筋トレ記録アプリ。ダーク高級感 × フィットネスのエネルギー感。

## フォルダ構成

```
筋トレアプリ/
├── 01_要件定義/
│   ├── 要件定義書.md          … 叩き台の要件（随時更新）
│   └── 元メモ_要件定義書.textClipping
├── 02_プロトタイプ/           … 携帯画面のHTMLモック
│   ├── index.html            … ← これをブラウザで開く
│   ├── css/style.css
│   └── js/
│       ├── config.js         … 接続先URL（スプレッドシート連携時に設定）
│       ├── api.js            … データ層（スプレッドシート / localStorage）
│       └── app.js            … 画面ロジック
├── 03_バックエンド/           … スプレッドシートDB (Google Apps Script)
│   ├── Code.gs               … スプレッドシートに貼るコード
│   └── README_デプロイ手順.md … デプロイ手順（5分）
└── README.md
```

## 使い方

**iPhone / どこからでも**: https://yuseikitagawa.github.io/kintore-app/
（Safariで開いて 共有 → ホーム画面に追加 でアプリとして使える）

**Macローカル**: `02_プロトタイプ/index.html` をブラウザでダブルクリックで開くだけ。ビルド不要。
下部タブで ホーム / 記録 / 履歴 / 統計 / メニュー の5画面を切り替えられる。
- 「記録」画面：セットの完了チェック・セット追加が動く
- 「メニュー」画面：部位別に種目を追加/削除、メニューへのON/OFF、複数メニュー（肩/背中/脚/胸/腕…）の追加・切替・名前編集ができる

## データ保存（スプレッドシートDB）

「入力したもの」は Google スプレッドシートに保存できる。
- 記録画面で「トレーニングを終了」→ 完了セットが **記録シート** に1行ずつ溜まる
- メニュー編集の変更 → **メニュー / 種目マスタシート** に反映
- 接続手順は `03_バックエンド/README_デプロイ手順.md`（`js/config.js` にURLを貼るだけ）

※ URL未設定でも動く。その場合はこの端末の localStorage に保存される（リロードしても残る）。
あとからURLを入れれば、以降がスプレッドシートに同期される。

## トレーニングスタイル

**部位別5分割ローテーション: 肩 → 背中 → 脚 → 胸 → 腕**

- ホームの「今日のメニュー」は **前回記録した部位の次** が自動で出る（今日すでに記録済みならその部位を継続）
- 記録画面はその部位の種目が最初から並び、**前回の重量・レップを初期値として引き継ぐ**
- 「＋ 種目を追加」は今日の部位の未追加種目から順に足される
- 履歴・統計（週間ボリューム、今週回数、連続日数、部位別バランス、自己ベスト）は記録データから自動集計。記録が無い間はサンプル表示
