# 筋トレ記録アプリ 要件定義書（叩き台 v0.1）

> 元メモがタイトルのみだったため、こちらで叩き台を作成。
> まずは携帯画面のHTMLモックで方向性を固め、反応を見ながら更新していく。

## 1. コンセプト

「続けたくなる」筋トレ記録アプリ。
数字を淡々と記録するだけでなく、**積み上げ（ボリューム・連続日数・自己ベスト）が一目で気持ちよく見える**ことを重視する。

- トーン：ダーク高級感 × フィットネスのエネルギー感
- 使う場面：ジムでスマホ片手に、その場でサッと記録

## 2. ターゲット

- 週2〜5回トレーニングする人
- 種目・重量・レップ・セットを記録し、前回比／自己ベストを確認したい

## 3. 画面構成（MVP）

| No | 画面 | 役割 |
|----|------|------|
| 1 | ホーム | 今日のメニュー、連続日数、週間ボリューム、クイックスタート |
| 2 | 記録 | トレーニング中のログ入力（種目→セット：重量×レップ） |
| 3 | 履歴 | 過去セッションの一覧、日付ごとの内容 |
| 4 | 統計 | 週次ボリューム推移、部位別、自己ベスト（PR）一覧 |
| 5 | メニュー | **部位別のメニュー編集**。種目マスタの追加/削除、メニューへの出し入れ、複数メニュー（肩→背中→脚→胸→腕の部位別5分割…）の追加・切替・リネーム |

## 4. 記録するデータ

- **セッション**：日付 / 所要時間 / 総ボリューム(kg) / 種目リスト
- **種目**：名称 / 部位(胸・背・脚・肩・腕・体幹) / セット配列
- **セット**：重量(kg) / レップ数 / 完了フラグ / （任意）RPE

## 5. 主要機能（MVP）

- [ ] トレーニングを開始／終了する
- [ ] 種目を追加し、セットを積み上げる（+セット）
- [ ] 前回の同種目の記録をその場で参照
- [ ] 自己ベスト更新をハイライト
- [ ] 連続トレーニング日数（ストリーク）
- [ ] 週間ボリュームのグラフ

## 6. あとで検討（Nice to have）

- テンプレート／ルーティン（PPL・部位分割など）
- 休憩タイマー
- 体重・体組成の記録
- Apple Health連携
- データのエクスポート／クラウド同期

## 7. 技術方針（プロトタイプ段階）

- 素の HTML + CSS + JS（ビルド不要、ダブルクリックで開ける）
- **データベースは Google スプレッドシート**（Apps Script をウェブアプリとして公開し窓口にする。無料・個人アカウント内で完結）
  - 記録＝1セット1行、メニュー/種目マスタ＝別タブ。スプレッドシート側から中身を直接確認・編集もできる
  - URL未設定でも localStorage で動作。設定後は同スキーマでスプレッドシートに同期
- まずは見た目と操作感の確認を優先。ロジックはサンプルデータで再現

## 8. 進め方

1. 携帯画面モックで4画面の雰囲気を確認 ← いまここ
2. 触りたい操作（記録フロー）を詰める
3. localStorage で実データ保存
4. 本実装（フレームワーク要否を判断）
